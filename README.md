# Pharmacy Management System

A web-based Pharmacy Management System built with **Laravel 12**. It provides a foundation for managing pharmacy operations — inventory, sales, prescriptions, suppliers, and role-based staff access — with real-time updates, PDF reporting, and QR code support.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Laravel 12 (PHP 8.2+) |
| UI / Reactivity | Livewire 3, Blade |
| Frontend build | Vite 7, Tailwind CSS 4 |
| JS libraries | jQuery, Select2, jQuery Mask, Laravel Echo, Pusher JS |
| Real-time | Laravel Reverb (WebSockets) |
| Authentication | Laravel Socialite (Microsoft OAuth) |
| Authorization | Spatie Laravel Permission (roles & permissions) |
| PDF | barryvdh/laravel-dompdf |
| QR Codes | simplesoftwareio/simple-qrcode |
| Cache / Queue backend | Redis (via Predis) |
| Database | MySQL |

## Requirements

- PHP **8.2+** with `pdo_mysql`, `gd`, `zip` extensions
- Composer 2.x
- Node.js 20+ and npm
- MySQL 8.x
- Redis (optional, used for cache/queue/broadcasting scaling)

## Installation

```bash
# 1. Clone the repository
git clone https://github.com/muaviayhasan/pharmacy.git
cd pharmacy

# 2. Install PHP dependencies
composer install

# 3. Install JS dependencies
npm install

# 4. Create your environment file
cp .env.example .env

# 5. Generate the application key
php artisan key:generate

# 6. Configure the database in .env, then create it
#    DB_DATABASE=pharmacy  DB_USERNAME=root  DB_PASSWORD=...

# 7. Run migrations
php artisan migrate

# 8. Build frontend assets
npm run build
```

## Running Locally

```bash
# Terminal 1 — application server
php artisan serve

# Terminal 2 — Vite dev server (hot reload)
npm run dev

# Terminal 3 — queue worker + Reverb WebSocket server (combined)
composer dev
```

The app will be available at <http://localhost:8000>.

## Configuration Notes

- **Database**: This project uses **MySQL** (not SQLite). Set `DB_*` values in `.env`.
- **Broadcasting**: `BROADCAST_CONNECTION=reverb`. Reverb credentials (`REVERB_APP_ID`, `REVERB_APP_KEY`, `REVERB_APP_SECRET`) are generated by `php artisan reverb:install`.
- **Redis**: `REDIS_CLIENT=predis` (pure-PHP client — no `phpredis` extension required).
- **Microsoft OAuth**: Set `MICROSOFT_CLIENT_ID`, `MICROSOFT_CLIENT_SECRET`, and `MICROSOFT_REDIRECT_URI` in `.env` to enable Socialite login.

## Testing

```bash
php artisan test
```

Tests run against a dedicated MySQL database (`pharmacy_test`) configured in `phpunit.xml`. Create it once with:

```sql
CREATE DATABASE pharmacy_test CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
